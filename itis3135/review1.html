CheckList

  1. Does the submission lead right to the page to be reviewed?: Yes. The link leads users right to the home page (index.html).
  2. Are there any spaces or upper-cases in the file/folder names, including scripts, images, etc.?: No. All website file/folder names and all other components utilize a proper naming format.
  3. Design:
    1. Does the page have sufficient contrast/font sizing?: Yes. Every page of the webpage utilizes a font that's easy to read in contrast with the white background. The size of the font is also large enough so most users will be able to see the information clearly.
    2. Does the page have site colors and fonts using a .css file?: Yes. The site implements the site colors and fonts using a .css file that's been linked in the head of each page.
    3. Does the page implement the CRAP Principles: ?
      1. Contrast: There's an excellent contrast of colors between the font and the backgrounds it's placed in front of. The different shades of blue with the white works well for users and looks aesthetically pleasing.
      2. Repetition: Repetition is displayed perfectly throughout the website. Fonts are consistent, and the same color scheme is used throughout each page of the site.
      3. Alignment: The website does a great job of having consistent alignment throughout its five pages. Even so, the website could truly benefit from having larger left and right margins for its text. The information on each page spans too far from edge to edge which can result in an uneasy reading experience for users.
      4. Proximity: Proximity isn't a problem for this website. Components that are associated with each other are placed in its own block on each page. This method makes the information organized and easy to read, in addition to being aesthetically pleasing.
  4. Does the page have the following within it?:
    1. Header: Yes.
    2. Main: Yes.
    3. Footer: Yes.
  5. Does the header have a site/brand header with text in an h1 tag?: The site header is there for the index page, but then is changed to the page name for the other pages. The site name; "Clearing the Air: Understanding Lung Cancer", should be consistent through navigation of all pages of the site. The h2 tag should state the page names instead.
  6. Does the main start with the page name as an h2, and not include the name of the site/brand?: Yes. The site utilizes h2 tags to write the page names at the start of the main. One area of critique could be that the page names in the nav bar and the h2 tag aren't consistent for every page of the site. For example, the "Screening Guidelines" page in the nav bar is written as "Lung Cancer Screening Guidelines" in the header. It would be best to keep consistent naming schemes for a more professional look and feel.
  7. Brand tagline?: There is no tagline for this project as it's not for a brand or company.
  8. Does the footer have the following components?:
    1. Menu for user's pages: Yes.
    2. HTML Validation: Yes.
    3. CSS Validation: Yes.
  9. Does the site have any specific requirements for the assignment?: Yes. All needed components are there on the site.
